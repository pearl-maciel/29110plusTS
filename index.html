Know the ISO/IEC 29110

ISO/IEC 29110

Software engineering — Lifecycle profiles for Very Small Entities (VSEs)

The ISO/IEC 29110 is an international standard for Software Engineering that define Lifecycles Profiles for Very Small Entities (VSEs). This standard was developed to improve the quality of the software product or service, at the same time they also wanted to improve the organization performance without excluding the use of diverse methodologies to help in the lifecycle of the software in development.

The ISO/IEC 29110 wants to give a standard that accommodate with the needs of a VSE. A Very Small Entity (VSE) is considered when the organization, department or project have 5 to 25 people working in it. This VSEs can be found in large corporations and they have a meaningful impact in it, so it is crucial for them to be at least well prepared and have an organized process to achieve their goal and don’t end up doing re-working for mistakes made by a bad software engineering.

There are “profiles” defined in the ISO/IEC 29110, and they are described as a set of one or more base standards to carry out a particular function by Professor Laporte, and those profiles are considered in one Group called Generic Profile in which the profiles are related by the composition of the process (activities, tasks, input, internal and output products), capacity level or both, the profiles that compose the Generic Profile are:

  • Entry/Starting: Aimed to the Small Entities (SE) with 6 people developing a project per month.
  • Basic: Aimed to SE developing only ONE project at thte time.
  • Intermediate: Aimed to SE developoing multiple projects inside the organization context.
  • Advanced : Aimed to SE that want to mantain and grow as an organization of Software Development independent and competitive.

The ISO/IEC 29110 defines two processes:

  • Project Management.
  • Software Implementation.

Security frameworks

TThis security frameworks were taken in consideration to improve the secuirty in the ISO/IEC 29110, which is the chosen base framework.

ISO/IEC 27001

The ISO/IEC 27001 is an international standard that take part in the series family of ISO/IEC 27000, in which recommendations to establish, implement, operate, monitor, review, maintain and improve an Information Security Managements System can be found.

ISO/IEC 27034

The ISO/IEC 27034 is an international standard that wants to assist organizations in integrating security throughout the life cycle of their applications. It offers guidance on information security, using security controls, to those involved in the System Development Life Cycle (or SDLC) of an application system.

Microsoft SDL

The Microsoft SDL is a secure software development process model in which best security practices are proposed and established in every step of the SDL (Software Development Lifecycle), then the structure of the model is based in an average software lifecycle that integrates security practices on each step.

Seven Touchpoints

The Seven Touchpoints are define, by G. McGraw, as a set of software security best practices, that can be implemented into the SDLC without making a fuzz in an organization, are based in good softer engineering and involve explicitly pondering the security state throughout the SDLC.

CORAS Methodology

CORAS is a model-based methodology for risk assessment, which performs a risk analysis that has a modeling language with tools specially designed for risk modeling. This methodology works in a defensive manner, which it is concerned with protecting what is already there.

OWASP Top 10 Mobile Security Risks

The OWASP Mobile Top 10 surged in 2014, with the firs list in which the vulnerabilities for mobile-server applications were identified. Then in 2016, the OWASP team decided to do a list focusing in the vulnerabilities of the Mobile applications and not the server vulnerabilities that could come with a service.

Improvements

The improvements to the base framework, the ISO/IEC 29110, were introduced in a task level after evaluating the explicit elements of every security framework considered.

The ISO/IEC 29110, considered as the base framework, was analyzed and the results of it was that the explicit element that could be modified or complemented by the security practices were the tasks and the output products.

The improvements to the base framework can be divided in two parts: 1) the integration of a security approach at the task level and output products, and 2) deployment packages to integrate task-level security practices that can be used in the development of secure mobile applications. The documentation generated in both parts can be found and downloaded in the Documentation section.

Security Practices

The tasks defined in the Base framework that were improved to integrate a security approach. The improved guide is available in the Documentation section in the ISO/IEC 29110 Improved subsection

Deployment Packages

A Deployment Package establish the procedure that explains how to do the tasks.

Security Practices.

The Security practices added to the Base framework complement the defined task with a security reinforcement. An example of the the security consideration added to a task is:

PM.1.3

Original - Identify the specific Tasks to be performed in order to produce the Deliverables and their Software Components identified in the Statement of Work. Include Tasks in the SI process along with verification, validation and reviews with Customer and Work Team Tasks to assure the quality of work products. Identify the Tasks to perform the Delivery Instructions. Document the Tasks.

Improved - Identify the specific Tasks to be performed in order to produce the Deliverables and their Software Components identified in the Statement of Work. Include:

  • Tasks in the SI process along with verification, validation and reviews with Customer and Work Team Tasks to assure the quality and security of work products.
  • Tasks of Security Requirements, which establish a minimum acceptable level of security and privacy quality.
  • Task to list assets that should be protected.
  • Task to identify the threats.
  • Task to identify information security risk.
  • Task to identify incidents.

Identify the tasks to perform the Delivery Instructions. Document the Tasks.

Deployment Packages.

The Deployment Packages describe procedures that can be used directly for the development of secure mobile applications. These are linked from the activities in the proposed improved ISO/IEC 29110 implementation guide.

Two main aspects were taken into consideration for the development of the deployment packages:

  1. The activities of Requirements, Design and construction of the software Implementation process.
  2. The considerations of a minimum level of security that a mobile application must have to guarantee its safe use from the point of view of 5 main aspects:
    Authentication.
    Authorization.
    Data Access.
    Data Storage.
    Data Transfer.
